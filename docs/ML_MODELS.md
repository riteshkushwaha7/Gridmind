# GRIDMIND тАФ ML Models

## LSTM тАФ Short-horizon Load & Demand Forecast

| Item                | Value                                                                                                                                                                                                                                                                              |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Inputs              | 192 timesteps ├Ч 14 features (48 h at 15-min res) + per-zone embedding (8d) concatenated to every timestep тЖТ effective input dim 22                                                                                                                                                  |
| Feature names       | `demand_kwh, base_load_kw, ev_load_kw, active_sessions, tariff_rate, solar_output_kw, battery_soc, temperature_c, cloud_cover_pct, hour_sin, hour_cos, dow_sin, dow_cos, is_weekend`                                                                                                |
| Outputs             | 16 timesteps ├Ч demand_kwh (4 h horizon, 15-min res). Confidence band = mean ┬▒ 1.96 ├Ч `residual_std` (computed on validation set, persisted as MLflow artifact)                                                                                                                       |
| Architecture        | `nn.Embedding(zones, 8)` тЖТ 2-layer `nn.LSTM(hidden=128, dropout=0.2)` тЖТ `LayerNorm` тЖТ `Dropout` тЖТ `Linear(hidden, 16)`                                                                                                                                                              |
| Training            | Huber loss (╬┤=1.0); Adam (lr 1e-3); `ReduceLROnPlateau(patience=3, factor=0.5)`; chronological 80/20 split; early stop patience 10                                                                                                                                                  |
| Schedule            | APScheduler (`backend/lstm/scheduler.py`) every 24 h тЖТ `train.run()` тЖТ register new version тЖТ POST `/lstm/reload` to inference server                                                                                                                                                |
| MLflow metrics      | `train_loss`, `val_loss`, `rmse_overall`, `residual_std`, `rmse_<zone_id>`, `mae_<zone_id>`                                                                                                                                                                                          |
| MLflow artifacts    | `scaler.pkl` (StandardScaler), `metadata.json` (per-zone metrics + residual std)                                                                                                                                                                                                     |
| Registry name       | `gridmind-lstm-load-forecast` (new versions auto-promoted to `Production`)                                                                                                                                                                                                          |
| Inference endpoint  | `POST :8010/lstm/forecast` body `{"zone_id":"Z01","horizon_hours":4}` тЖТ `ForecastResponse` (forecast points + permutation feature_importance + rmse_last_eval)                                                                                                                       |

## PPO тАФ Charger Schedule Controller

| Item                | Value                                                                                                                                                                                                                                                                                                |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Observation space   | 70 floats: `forecast(16) тКХ tariff(1) тКХ feeder_headroom_pct(10) тКХ active_evs(10) тКХ avg_soc(10) тКХ battery_soc(10) тКХ solar_output_kw(10) тКХ [hour_sin, hour_cos, is_weekend]` (matches `PPO_OBS_DIM` in `backend/config.py`)                                                                              |
| Action space        | `Box(low=0, high=1, shape=(10,), float32)` тАФ normalised charging-power setpoint per zone                                                                                                                                                                                                              |
| Reward              | `R = тИТ(╬▒┬╖cost_inr + ╬▓┬╖peak_load_norm┬╖100) + ╬│┬╖energy_delivered_norm┬╖100`                                                                                                                                                                                                                              |
| Reward weights      | `╬▒ = 0.5` (cost), `╬▓ = 0.3` (peak penalty), `╬│ = 0.2` (energy delivered) тАФ overridable via `PPO_REWARD_*` env vars                                                                                                                                                                                     |
| Safety constraint   | Hard cap `P_final = min(P_AI ┬╖ capacity, P_max_feeder)` applied in `ppo/safety.py` before every action reaches the env or the live grid. Rule-based **proportional fallback** activates when `feeder_headroom_pct < SAFETY_HEADROOM_THRESHOLD_PCT` (default 10 %): each EV gets `headroom_kw / active_evs ┬╖ 0.8`. Every override writes a row to InfluxDB measurement `safety_events`. |
| Training            | Stable-Baselines3 PPO, `MlpPolicy`, `net_arch=[256,256]`, `n_steps=2048`, `batch_size=64`, `n_epochs=10`, `lr=3e-4`, `clip_range=0.2`, `ent_coef=0.01`. Scratch run = 500 000 timesteps; online-update job = 50 000 timesteps every 6 h                                                              |
| MLflow metrics      | `mean_reward`, `std_reward`, `mean_episode_cost_inr`, `mean_energy_kwh`, `peak_reduction_pct`, `safety_violation_rate`                                                                                                                                                                                |
| Registry name       | `gridmind-ppo-scheduler` (SB3 zip artifact; auto-promoted to `Production`)                                                                                                                                                                                                                            |
| Inference endpoint  | `POST :8011/ppo/schedule` тЖТ `ScheduleResponse` (per-zone setpoint + safety_capped/fallback_active flags + reasoning string)                                                                                                                                                                            |

## Clustering тАФ Candidate Zone Scoring

| Item            | Value                                                                                                                                                                                                                                                                                                                                                  |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Inputs          | Per-zone, 30-day rollups from InfluxDB: `avg_demand_kwh_daily, demand_growth_rate_pct, peak_demand_kw, demand_variance, grid_headroom_avg_pct, grid_headroom_min_pct, existing_charger_count, charger_utilization_pct, avg_queue_length, ev_adoption_growth_rate`                                                                                       |
| Algorithms      | StandardScaler тЖТ KMeans(`n_clusters=CLUSTERING_K=3`) for HIGH/MEDIUM/LOW priority; DBSCAN(`eps=0.8, min_samples=2`) for outlier flagging                                                                                                                                                                                                                |
| Scoring formula | `Score(z) = ╬▒┬╖D(z) + ╬▓┬╖G(z) тИТ ╬│┬╖C(z)`, all components min-max normalised; `╬▒=0.4, ╬▓=0.3, ╬│=0.3`. `D = 0.6┬╖norm(avg_demand) + 0.4┬╖norm(growth_rate)`; `G = 1 тИТ norm(grid_headroom_avg_pct)`; `C = norm(existing_charger_count)`. Final score re-normalised to `[0, 1]`; top 3 = HIGH, next 4 = MEDIUM, bottom 3 = LOW.                                   |
| Recommendation  | `chargers_needed = ceil(avg_demand ┬╖ growth ┬╖ forecast_horizon / avg_kwh_per_charger_day)`; charger type = `DC_Fast_50kW` if headroom < 30 % else `L2_AC_22kW`; cost = chargers ├Ч `CHARGER_COST_INR_LAKHS[type]`                                                                                                                                          |
| Schedule        | APScheduler 7-day trigger inside `clustering/inference.py`; also POST `/clustering/replan` for on-demand recompute                                                                                                                                                                                                                                       |
| MLflow          | The clustering pipeline is stateless / re-fit each pass; no model registry write. Audit row goes to Postgres `audit_log`; cache-warm copy lives at Redis key `clustering:latest_ranking`.                                                                                                                                                                |
| Endpoint        | `GET :8012/clustering/zones/ranking` тЖТ `RankingResponse` (zones sorted by score, with priority + recommendation + silhouette)                                                                                                                                                                                                                          |
